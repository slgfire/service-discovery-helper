# service-discovery-helper v2.0 — Design Spec

## Overview

Modernize SDH from a bare-bones CLI tool into a production-ready service while keeping the core forwarding logic unchanged. All changes land on a `v2.0` branch; `master` stays at v1.x.

## 1. Signal Handler (Graceful Shutdown)

**Problem:** SDH has no way to shut down cleanly. `kill` leaves threads dangling and stat files may not be flushed.

**Design:**
- Change `do_exit` from `int` to `volatile sig_atomic_t` (in both `sdh-proxy.c` and `sdh-proxy.h`) to prevent the compiler from optimizing out the check in `start_listening()`
- Register handler for SIGTERM and SIGINT
- Handler ONLY sets `do_exit = 1` — no other calls (async-signal-safety)
- Main thread detects signal (returns from `pthread_join` or checks flag), then iterates `iface_data[]` calling `pcap_breakloop()` on each `pcap_t` from outside the handler
- Add `do_exit` check to `timer_purge_old_entries_loop`'s `while(1)` so the purge thread also exits cleanly
- Main thread joins ALL threads (worker threads + purge thread)
- After all threads exit, flush stat file one final time if logging is enabled
- Clean up allocated memory (pcap handles, filter string, iface/port lists)

**Files changed:** `sdh-proxy.c`, `sdh-proxy.h`, `timer.c`

## 2. Thread Safety for Counters

**Problem:** `pkt_rx`, `pkt_tx`, `pkt_drop` (int64_t) and `pkt_stats[]` (unsigned short) are written from multiple threads without synchronization. Race conditions on counters.

**Design:**
- Replace direct increments with GCC atomic builtins: `__sync_fetch_and_add(&pkt_rx, 1)`
- Applies to: `pkt_rx`, `pkt_tx`, `pkt_drop`, `pkt_stats[]`, `iface_data[i].num_packets`
- No mutex needed — these are simple add-one operations on aligned types
- Change `pkt_stats[]` from `unsigned short int [65535]` to `uint32_t [65536]`:
  - `uint32_t` ensures 4-byte alignment for reliable atomics on all targets (armhf has no native 2-byte atomic instructions)
  - Array size 65536 fixes pre-existing off-by-one (UDP ports range 0–65535 inclusive)
- Update stat file format string accordingly (`%u` instead of `%ld`)

**Files changed:** `sdh-proxy.c`, `sdh-proxy.h`

## 3. Centralized Logging

**Problem:** Logging is scattered `printf`/`fprintf` calls. No log levels. Not suitable for daemon operation.

**Design:**
- New file: `log.c` / `log.h`
- Function: `void sdh_log(int level, const char *fmt, ...)`
- Levels: `SDH_LOG_ERROR`, `SDH_LOG_INFO`, `SDH_LOG_DEBUG`
- Two output backends, selectable at runtime:
  - **stdout** (default): formatted with timestamp prefix `[YYYY-MM-DD HH:MM:SS] LEVEL: message`
  - **syslog**: uses `vsyslog()` with `LOG_DAEMON` facility — NO timestamp prefix (syslog adds its own)
- `SDH_LOG_DEBUG` messages only emitted when debug mode is on
- Replace all existing `printf`/`fprintf` calls with appropriate `sdh_log()` calls
- `sdh_log_init(int use_syslog)` called once at startup
- `sdh_log_cleanup()` called at shutdown (closes syslog if open)

**Files added:** `log.c`, `log.h`
**Files changed:** `sdh-proxy.c`, `timer.c`

## 4. Config File

**Problem:** All settings require CLI flags. Complex setups need long command lines.

**Design:**
- New file: `config.c` / `config.h`
- INI-style format with `#` comments (consistent with existing ports/interfaces files):

```ini
[interfaces]
eth0.100
eth0.101
# or use: auto

[ports]
27015-27020    # Source Engine
6112           # Warcraft 3

[settings]
rate_limit = yes
rate_limit_timeout = 750
log_stats = yes
syslog = no
debug = no
```

- Parser reads sections, populates the same global variables the CLI parser uses
- **Bounds checking:** Parser rejects input if `num_ifaces >= MAX_IFACES` or `num_ports >= MAX_PORTS` with an error message and non-zero return
- New CLI flag: `-c <path>` to specify config file
- **Auto-discovery:** If no `-c` given and no other flags given, look for `/etc/sdh-proxy.conf`. If not found, fall back to requiring CLI flags (existing behavior)
- **`argc` guard:** Remove the existing `argc < 3` early-exit check. Validation happens after parsing: "if num_ports == 0 || num_ifaces == 0, exit" (already exists)
- **Override order:** Config file loaded first, then CLI flags override. This means `-c config.conf -d` loads the config but forces debug on
- `[interfaces]` section: if a line is `auto`, set `use_all_interfaces = 1`
- `[ports]` section: same format as existing port files (single port or range `XXXXX-XXXXX`)
- `[settings]` section: key = value pairs, values are `yes`/`no` for booleans, integers for timeouts
- **`rate_limit_timeout` conversion:** `pkt_timeout_s = value / 1000; pkt_timeout_us = (value % 1000) * 1000;` — matches existing CLI code

**Parsing approach:** Simple line-by-line parser. Track current section with an enum. No external library.

**Files added:** `config.c`, `config.h`
**Files changed:** `sdh-proxy.c` (add `-c` flag handling, remove `argc < 3` guard, call config parser before CLI override)

## 5. Systemd Service File

**File:** `sdh-proxy.service`

```ini
[Unit]
Description=Service Discovery Helper - UDP broadcast forwarder
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
ExecStart=/usr/local/bin/sdh-proxy -c /etc/sdh-proxy.conf
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

**Install instructions** added to README:
```bash
sudo make install
sudo systemctl daemon-reload
sudo systemctl enable --now sdh-proxy
```

## 6. Makefile Changes

**Updated `all` target:**

```makefile
CC ?= gcc

all:
	$(CC) -Wall -g -std=gnu99 -o sdh-proxy sdh-proxy.c timer.c log.c config.c -lpthread -lpcap
```

**New targets:**

```makefile
PREFIX ?= /usr/local
SYSCONFDIR ?= /etc
SYSTEMD_UNIT_DIR ?= /lib/systemd/system

install: all
	install -D -m 755 sdh-proxy $(DESTDIR)$(PREFIX)/bin/sdh-proxy
	install -D -m 644 sdh-proxy.service $(DESTDIR)$(SYSTEMD_UNIT_DIR)/sdh-proxy.service
	test -f $(DESTDIR)$(SYSCONFDIR)/sdh-proxy.conf || install -D -m 644 sdh-proxy.conf.example $(DESTDIR)$(SYSCONFDIR)/sdh-proxy.conf

uninstall:
	rm -f $(DESTDIR)$(PREFIX)/bin/sdh-proxy
	rm -f $(DESTDIR)$(SYSTEMD_UNIT_DIR)/sdh-proxy.service
	# Config file intentionally not removed (preserves user changes)
```

## 7. Dockerfile

**File:** `Dockerfile` (multi-stage build for correct multi-arch support)

```dockerfile
FROM debian:bookworm-slim AS builder
RUN apt-get update && apt-get install -y --no-install-recommends gcc make libpcap-dev libc6-dev && rm -rf /var/lib/apt/lists/*
WORKDIR /src
COPY . .
RUN make

FROM debian:bookworm-slim
RUN apt-get update && apt-get install -y --no-install-recommends libpcap0.8 && rm -rf /var/lib/apt/lists/*
COPY --from=builder /src/sdh-proxy /usr/local/bin/sdh-proxy
COPY sdh-proxy.conf.example /etc/sdh-proxy.conf
ENTRYPOINT ["sdh-proxy", "-c", "/etc/sdh-proxy.conf"]
```

- Multi-stage: builder compiles per target arch via `docker buildx`, final image only contains runtime deps
- Requires `--net=host` to access host network interfaces
- Config mountable: `-v /path/to/my.conf:/etc/sdh-proxy.conf`

**CI addition to release workflow:**
- Build and push to GitHub Container Registry (`ghcr.io/slgfire/service-discovery-helper`)
- Tagged with version (e.g., `v2.0`, `latest`)
- Multi-arch manifest (x86_64, arm64, armhf) via `docker buildx build --platform`

## 8. Release Workflow Changes

- Add Docker build+push job to `release.yml` (only for `v2*` tags)
- Multi-arch Docker image via `docker buildx build --platform linux/amd64,linux/arm64,linux/arm/v7`
- Binary releases continue as before for all tags
- **Add `sdh-proxy.conf.example` to release tarball** packaging step

## 9. Branch Strategy

- Create `v2.0` branch from current `master`
- All v2.0 work happens on `v2.0` branch
- CI workflow triggers on `v2.0` branch pushes in addition to `master`
- Release workflow triggers on `v2*` tags (e.g., `v2.0`, `v2.0.1`)
- `master` remains the stable v1.x line
